# KS-S0-006 · source_manifest_hash 生成

## 1. 任务目标
- **业务**：把 clean_output 整体冻结成可哈希、可比对的快照，作为下游 compile_run_id 的源。
- **工程**：实现 `generate_manifest.py`，输出 `source_manifest.json`：列出每个 canonical 文件路径 + sha256 + size + mtime；顶层 `manifest_hash` = 全部条目 hash 链。
- **S gate**：S0；为 §2 governance_common_fields 中的 `source_manifest_hash` 提供唯一来源。
- **非目标**：不改任何被 hash 的文件。

## 2. 前置依赖
- KS-S0-001（基线对齐后才有意义）

## 3. 输入契约
- 读：`clean_output/nine_tables/*.csv`、`clean_output/candidates/**/*.yaml`、`clean_output/play_cards/**/*`、`clean_output/runtime_assets/**/*`、`clean_output/registers/*.csv`
- 不读：audit/（避免循环）

## 4. 执行步骤
1. 实现脚本：递归列出 canonical 路径（白名单驱动，避免误把 audit 计入）
2. 对每个文件计算 sha256
3. 顶层 hash = `sha256(sorted entries 序列化)`
4. 落 `clean_output/audit/source_manifest.json`：`{manifest_hash, generated_at, entries:[{path, sha256, size}]}`
5. `--verify` 模式：重算并与已落盘 manifest 对比
