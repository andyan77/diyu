# KS-S0-003 · ECS 密钥轮换 + 移出仓库

## 1. 任务目标
- **业务**：消除 SSH/PG/Qdrant 凭证被仓库历史泄露的风险。
- **工程**：把所有密钥迁移到环境变量；旧 key 在 ECS 侧吊销；轮换记录留痕。
- **S gate**：S0。
- **非目标**：不实现新的密钥托管系统；不改业务代码。

## 2. 前置依赖
- 无（DAG 起点）

## 3. 输入契约
- 读：`ECS_AND_DATA_TOPOLOGY.md` 中的凭证位置（仅用于定位，不外泄）
- 环境变量名约定：`ECS_SSH_KEY_PATH`, `PG_USER`, `PG_PASSWORD`, `PG_HOST`, `PG_PORT`, `QDRANT_URL_STAGING`, `QDRANT_API_KEY`

## 4. 执行步骤
1. `git grep -E "(password|api_key|secret|sk-)" -- ':!docs'` 列出所有命中
2. 把每个命中替换为 `os.environ["..."]` 引用
3. 写 `.env.example` 列全部需要的变量名（不含值）
4. `.gitignore` 加 `.env`
5. 在 ECS 侧吊销旧 SSH key、旋转 PG 密码、旋转 Qdrant API key
6. 写 `clean_output/audit/key_rotation_log_KS-S0-003.md`：旋转时间、影响范围、新 key 指纹（不写明文）
7. `git log -p` 检查历史中是否有泄露提交；若有，记录到日志（不强制 history rewrite）
