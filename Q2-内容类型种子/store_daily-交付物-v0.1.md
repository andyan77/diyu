## 本文件是什么（核心定义，防止脱离索引被误读）

- **交付物 ≠ 字段 schema**。交付物是一份"**玩法库 + 元素库 + 红线**"，是 AI 生产这种内容时可调用的**表达资源包**。
- **模板 ≠ 文字填空模板**。每张玩法卡里的「30 秒分镜参考」就是模板——**拍摄者照着能拍出来的执行卡片**，含画面 + 声音 + 动作 + 节奏。
- **AI 生产时不是"填字段"**，而是"**从玩法库里选一张卡片并展开**"。

> 定义来自用户 2026-04-13 裁决。详见 [_index.md](_index.md) §0。

## 本文件与 batch 1/2 的关键差异：两者均可类的边界

### 1. store_daily vs personal_vlog 的硬边界（用户 2026-04-14 裁决）

batch 1 的 `personal_vlog` 是"**人作为叙事主体**"——镜头核心是创作者本人的生活、情绪、时间流。本 ContentType 是"**门店作为叙事主体**"——镜头核心是门店空间的运营节奏、商品与员工的互动、消费者与门店的关系。

**硬边界三要素自检（三选二必达）：**

| 要素 | 判定标准 |
|---|---|
| ① 店铺空间 | 画面中至少有 50% 时间出现**门店可辨识空间**（货架 / 镜子 / 试衣间 / 收银台 / 门口 / 后仓 / 吊牌墙）|
| ② 商品道具 | **衣架 / 面料 / 吊牌 / 蒸汽熨斗 / 镜子 / 纸袋** 等服装门店特有物件至少出现 3 次以上 |
| ③ 员工工作节点 | 至少包含一个**真实零售运营动作**：补货 / 熨烫 / 折叠 / 点钞 / 陈列 / 接待 / 打包 / 盘点 |

**三选二必达 → 归 `store_daily` ; 只命中一个或零个 → 应归 `personal_vlog` 或 `daily_fragment`。**

### 2. 双角色定位（引流 vs 接客）

`store_daily` 同时服务 Q1 第一层（引流）和第二层（接客）——玩法本体相同，差异是执行情境。每张进阶 / 突破性玩法卡含「**适用场景**」标签标注引流向 / 接客向 / 两者通用。

### 3. 和 `founder_ip` 的区别

`founder_ip` 的叙事重心是"创始人这个人"（决策 / 代价 / 性格）。本 ContentType 的叙事重心是"门店这个空间"（秩序 / 节奏 / 运营纪律）。即使老板本人出镜，只要观众记住的是"这家店"而不是"这个老板"，就归 `store_daily`。

## A. 北极星

**观众看完后，心里不是"这家店很漂亮"或"这老板很忙"，而是"这家店是活的，而且有人在认真过每一天——有秩序、有节奏、有标准、有温度，我想走进去看看"。**

> **三份原话备查：**
> - 通用类.md：这家店是活的，而且有人在认真过每一天
> - 通用类compass.md：a recognizable human at the center — 门店不是产品目录，而是有人在经营的舞台
> - 通用类deep-research-report.md：这家店是真实在运转的、有人味、有标准——信任在"看见流程"而不是"听见承诺"中建立

## B. 元素库（原子层）

### B1 视觉元素
- 门店广角空镜（开门前 / 闭店后 / 清晨阳光穿过货架）
- 手部特写（折叠衣服 / 挂回衣架 / 熨烫 / 点钞 / 整理吊牌）
- 俯拍桌面（打包台 / 收银台 / 盘点台 / 分类的三堆衣服）
- 固定机位中景（收银台 / 门口 / 试衣间外 / 货架前）
- 镜面反射（试衣间镜 / 收银台镜 / 橱窗玻璃——不直接拍人脸）
- 模特 / 人台特写（更换、整理、换装瞬间）
- 第一视角手持跟拍（员工走位 / 补货路线 / 闭店路线）
- 物件痕迹（镜子上的手印 / 试衣间地垫 / 挂钩上没带走的衣服 / 外卖咖啡杯）
- 衣架 / 吊牌 / 纸袋 / 蒸汽熨斗作为"门店生命体征"的证据物

### B2 听觉元素
- 卷帘门 / 开门铃 / 门锁声
- 衣架滑动声 / 衣架碰撞桌面声
- 布料摩擦声 / 拉链声 / 蒸汽喷出声
- 剪刀剪吊牌声 / 胶带封袋声 / 纸张声
- 收银台抽屉拉开 / 点钞声
- 顾客脚步 / 门口街声 / 雨声 / 外卖车声
- 极少口播 / 不使用 BGM 或仅极弱环境音 BGM
- 半句对话片段（"有小一码吗" / "我再看看" / "谢谢"）

### B3 人物元素
- 只露手（手部是门店日常最强的"信任证据"）
- 只露背影 / 侧影 / 局部（员工在做事，不看镜头）
- 店员本人第一视角
- 顾客的手 / 脚 / 背影（隐私保护 + 关系感）
- 跨代际同框（老板 + 老员工 / 店长 + 新店员）
- 熟客配合拍摄（避免偷拍陌生顾客）

### B4 场景元素
- 门店前台 / 收银台 / 货架
- 试衣间（帘子 / 挂钩 / 地垫 / 外面等待区）
- 后仓 / 打包台 / 盘点区
- 门口 / 街边 / 橱窗内侧
- 仓库 / 进货区
- 镜前区域（镜子 + 地板 + 穿搭候选）
- 员工休息角（后台桌子 / 外卖咖啡 / 手机 / 工作清单）

### B5 情绪元素
- 秩序感（重复动作 / 标准化流程 / 归位）
- 节奏感（准备→忙乱→空档→收束的起伏）
- 克制感（不解释 / 不表演 / 不碎碎念）
- 温度感（随手的细节 / 顾客犹豫时的等待）
- 粗粝感（退货 / 起球 / 瑕疵的坦然展示）
- 稳定感（每一天都在重复同一批动作）
- **避开**："假装忙" / "假装治愈" / "假装爆单"

### B6 文字元素
- 极少字幕（只标关键动作 / 时间戳 / 1-2 句结尾留白金句）
- 手写牌 / 纸条（"为什么退" / "哪里坏" / "怎么救"）
- 订单号打码的步骤标签（打包流程）
- 完全无结论 / 完全无 CTA（不写"欢迎光临" / "关注我"）
- 反鸡汤留白句（"我们卖的不是衣服，是你走进来那一刻的确定感"）

### B7 时间元素
- 单一时段（只拍开门前 30 分钟 / 只拍闭店后 30 分钟）
- 情绪分段（准备 / 忙乱 / 空档 / 收束——不按钟点走）
- 一天流逝（清晨光线 → 正午顾客 → 傍晚收束）
- 重复动作蒙太奇（同一动作在一天里的 N 次）
- 静默断点（3-5 秒不动的空镜）

### B8 反差元素
- 完美陈列 vs 真实运营背后的混乱
- 前台客服语气 vs 后仓私下动作
- 开店前的安静 vs 闭店后的安静（同构但意义完全不同）
- 新品光鲜 vs 退货堆
- 标准化 SOP vs 顺手的个性化（折衣服的独特手法）

### B9 隐喻元素
- 门店 = 有呼吸频率的空间
- 衣架声 / 蒸汽声 = 门店的"心跳"
- 镜子 = 顾客身份切换的装置
- 试衣间 = 第三空间（私密 / 自我重塑）
- 退货堆 = 品质态度的证据
- 关灯的一瞬 = 一天的句号

## C. 玩法库（核心交付物）

### C1 主流玩法（默认禁用 / 低优先）

> 本档玩法是门店内容**最容易陷入的套路**——"开门流水账 / 新品陈列广播 / 打卡式门店美学"。这些玩法**不是不能拍**，而是观众已经免疫——没有信息密度、没有张力、没有门店的性格。笛语平台对 store_daily 的 C1 档**默认禁用**。

#### 玩法卡 #C1-1 —— "开门关门流水账"

| 字段 | 内容 |
|---|---|
| 档位 | 🚫 主流 / 默认禁用 |
| 一句话定义 | 开门 → 拖地 → 开灯 → 整理衣架 → 收银 → 下班锁门，一条 BGM 从头铺到尾，字幕"开店啦 / 下班啦 / 今天也要加油"|
| 北极星映射 | ❌ 只看到流程、看不到"人"、看不到"门店的性格"——像打卡记录，不像有呼吸的空间 |
| 表达元素清单 | B1 广角扫店 / B2 BGM 盖过环境音 / B1 快剪 / B6 鸡汤字幕 / B5 假装治愈 |
| 服装行业钩子 | ❌ 服装只是背景装饰，门店专业能力完全没被看见 |
| 实拍可行性 | 🟢 易拍，但**门槛低 → 同质化 → 被直接划过** |
| 30 秒分镜参考 | （主流玩法不写分镜，避免被复用） |
| AI 辅助度 | ⚠️ AI 最容易模仿 ; 也最容易被观众识别为"千店一面"|
| 来源 | 通用类.md §2 主流玩法 1 + 深度研究 §store_daily 主流玩法 1（合流） |
| 备注 | **服装行业特别警告**：这个模板在服装行业的危害不是"没流量"，而是让观众对"这家店"失去任何具体印象 |

#### 玩法卡 #C1-2 —— "新品到店陈列 + 价格广播"

| 字段 | 内容 |
|---|---|
| 档位 | 🚫 主流 / 默认禁用 |
| 一句话定义 | 镜头横移扫一排新品 + 字幕标价 / 标折扣 + 口播"春夏上新 / 本周特价" |
| 北极星映射 | ❌ 像把线下价签搬到线上——观众会直接归类为广告而跳过 |
| 表达元素清单 | B1 横移扫货架 / B2 口播 + 促销 BGM / B6 价格字幕 / B4 单一货架场景 |
| 服装行业钩子 | ❌ 服装在此被降维成 SKU，不是故事 |
| 实拍可行性 | 🟢 易拍 |
| 30 秒分镜参考 | （主流玩法不写分镜，避免被复用） |
| AI 辅助度 | ⚠️ AI 一键生成 |

#### 玩法卡 #C1-3 —— "打卡式门店美学空镜"

| 字段 | 内容 |
|---|---|
| 档位 | 🚫 主流 / 默认禁用 |
| 一句话定义 | 只拍陈列、橱窗、试衣间、灯光，配慢推镜头 + 滤镜 + 轻音乐 + 无人物 |
| 北极星映射 | ❌ 有审美没有生命体征——门店被拍成静物，像酒店探店而不是门店日常 |

### C2 进阶玩法（少数高手在用）

#### 玩法卡 #C2-1 —— "看不见的劳动 / 营业的一天只拍支持动作"

| 字段 | 内容 |
|---|---|
| 档位 | ⭐ 进阶 |
| 适用场景 | **引流 / 接客两者通用**（引流层：展示"专业感" ; 接客层：让进店顾客产生"这家店在认真做事"的信任） |
| 一句话定义 | 不拍卖货，只拍支持卖货的动作：拆箱 / 熨烫 / 对尺码 / 挂回错位衣架 / 接待前 30 秒整理镜面 / 闭店后核对明天要上的款。把"门店专业感"从结果挪到过程 |
| 北极星映射 | ✅ 观众开始理解——一家店为什么值得信任，不只是因为装修好看，而是因为有人在背后一直修正细节 |
| 表达元素清单 | B1 手部劳动 / B1 重复动作 / B1 局部特写 / B2 门店环境音 / B3 只露手 / B6 极少字幕 / B7 时间流逝 |
| 30 秒分镜参考 | **0-5 秒：** 手部特写从纸箱里抱出叠好的新衣 + 纸箱开封声 \| **5-12 秒：** 俯拍台面，手把衣服一件件摊开对尺码 + 吊牌翻动声 \| **12-20 秒：** 切到蒸汽熨斗喷一下领口 + 喷汽声 + 手捏领子 \| **20-27 秒：** 走到货架，把挂错位置的一件移到对的色系组 \| **27-30 秒：** 镜头停在整齐的一排新品 2 秒，无字幕 |
| 服装行业钩子 | ⭐⭐⭐ 服装门店的"看不见的劳动"有极强的**具象证据**——每个动作都是可被信任的证据（熨烫 = 在意品相 / 对尺码 = 不糊弄 / 挂回错位 = 不容许混乱）|
| 实拍可行性 | 🟢 1 人 1 手机（本来就在做的事顺手拍）/ 预算 0 / 不依赖演员 / 4 小时内完成 |
| AI 辅助度 | ✅ 能做：镜头清单、重复动作节拍建议、字幕极简建议 ; ❌ 不能做：替代真实运营动作 |
| 来源 | 通用类.md §2 进阶玩法 1 + compass "open the store with me / merchandising as a boutique owner"（合流）|
| 备注 | Instagram 上 boutique owner 已形成稳定分支——"open the store with me / just opening and closing / merchandising" 是国际成熟玩法模板 |

| 字段 | 内容 |
|---|---|
| 档位 | ⭐ 进阶 |
| 适用场景 | **引流为主**（塑造门店气质，让观众记住"这家店的性格"）|

| 服装行业钩子 | ⭐⭐⭐ 服装门店天然有"情绪光谱"——面料的重量感、灯光的暖度、镜前的停顿时间，都是可拍的情绪证据 |
| 实拍可行性 | 🟢 主要靠观察和剪辑顺序 / 预算 0 / 不依赖演员 |
| AI 辅助度 | ✅ 能做：情绪段落建议、光线抓取建议、字幕留白建议 ; ❌ 不能做：替代"今天的门店实际上是什么感觉"|
| 来源 | 通用类.md §2 进阶玩法 2 + compass "founder diaries / boutique day-in-the-life"（合流）|
| 备注 | 很多服装品牌的 founder diaries 有效，不是展示工作量，而是让人感到"**这是一种生活**"|

#### 玩法卡 #C2-3 —— "打包即内容 / packing orders"

| 字段 | 内容 |
|---|---|
| 档位 | ⭐ 进阶 |
| 适用场景 | **两者通用**（引流层：展示品牌气质 ; 接客层：让店内顾客看见打包标准增加信任） |
| 一句话定义 | 把门店最重复、最能体现"控标准"的一段变成内容——拣货 / 核对 / 折叠 / 封袋 / 贴单 / 加入小卡。不是说"我们服务好"，而是把"服务好"拍给你看 |
| 北极星映射 | ✅ 打包天然具备"流程美学 + 满足感 + 标准可视化"——服务质量从承诺变成可观看的证据 |

### C3 突破性玩法（笛语平台主推）

> 本档是笛语平台对 store_daily 的核心交付物。**门店日常的突破性不在"拍得多美"，而在"把线下不可见的专业变成线上可见的证据链"**。AI 生产 store_daily 时应该**优先**从 C3 取卡。共 9 张。

#### 玩法卡 #C3-6 —— "衣架摄像机 / Rack Walk"

| 档位 | 🌟 突破性 |

| 北极星映射 | ✅ 观众记住的是一种"人情与选择"的微小戏剧，而不是促销信息 |

| 服装行业钩子 | ⭐⭐⭐⭐ 服装成为**主角**——门店的专业（选款 / 判断 / 推荐）被自然呈现 ; 这是把"商品"变成"叙事主体"的最佳玩法 |
| 实拍可行性 | 🟡 需要朋友帮忙拿机或固定绑带（≤ 200 元）/ 不依赖专业 / 4 小时内完成（场景都在店内） |

#### 玩法卡 #C3-7 —— "试衣间独白 / Store as Theater"

| 字段 | 内容 |
|---|---|
| 档位 | 🌟 突破性 |
| 适用场景 | **引流为主**（戏剧化、有艺术电影质感，最容易形成平台爆点）|

## D. 服装行业垂直机会

服装门店做 `store_daily` 的**最大优势**不是"门店漂亮能拍"，而是**服装门店本身就是一个天然的节奏空间**——衣架 / 镜子 / 帘子 / 尺码 / 蒸汽 / 叠衣 / 模特更换，这些都天然有视觉与声音层次。

### 三个未被开发的角度

1. **"把线下不可见的专业变成线上可见的证据"**（C2-1 看不见的劳动 / C2-3 打包即内容 / C3-4 库存解剖室 / C3-9 ASMR）——零售运营的本质是"把总部策略变成门店每一天的稳定执行"，而多门店最难的是一致性。`store_daily` 可以**对外展示门店生命力、对内展示运营纪律**，双重任务一次完成
2. **"服装门店独有的声音世界"**（C3-1 纯沉默 / C3-9 ASMR）——衣架声 / 布料摩擦 / 蒸汽喷出 / 试衣间帘子，是别的零售业态没有的"声音素材库"，切入 ASMR 赛道几乎无人竞争
3. **"门店作为戏剧空间"**（C3-3 关系碎片 / C3-5 人台视角 / C3-7 试衣间独白 / C3-8 镜面对话）——服装店有**其他零售没有的心理空间**（试衣间 = 第三空间 / 镜子 = 身份切换装置 / 人台 = 被穿着的观察者），这是服装行业独有的艺术化内容空间

### 行业趋势支撑

- **EGC（员工生成内容）研究**：员工生产的社媒内容会影响消费者对"品牌公民行为"与"专业性"的感知，进而提升购买意向与口碑
- **Zhihu 300 账号分析**：街拍属流量转换逻辑、店铺属纯售卖逻辑、生活 KOL 属粉丝逻辑——能突破的账号都有"一个可被辨识的人在中心"
- **#packingorders 趋势**：TikTok 上出现"付费看打包"现象，履约过程内容化是全球级趋势
- **ASMR 赛道缺口**：服装行业 ASMR 潜力被严重低估，AI ASMR 账号已证明 5 天 10 万粉的爆发力，**尚无服装门店系统化应用**

## E. 红线（一定不行的玩法）

| # | 红线 | 来源 |
|---|---|---|
| E1 | **偷拍顾客脸 / 录到私人对话 / 任何可识别身份信息** —— 信任会瞬间反噬，且可能触发平台投诉与法律风险 | 深度研究 §store_daily 红线 1 |
| E2 | **"假装爆单 / 假装排队 / 假装热闹"** —— 门店场景里一旦被识破，观众会把你归类为"演"，信任无法挽回 | 深度研究 §store_daily 红线 2 |
| E3 | **把门店日常拍成硬广** —— 只剩促销信息时，失去"门店作为真实生活空间"的最大优势 | 深度研究 §store_daily 红线 3 |
| E4 | **只有漂亮空镜，没有真实动作** —— 像摆拍 showroom，不像门店日常 | 通用类.md §2 红线 1 |
| E5 | **全片强调"我们很忙"却看不出具体忙什么** —— 变成"忙碌的表演"，观众直接跳过 | 通用类.md §2 红线 2 |
| E6 | **为了热闹强行演顾客互动**，假笑假聊一眼出戏 | 通用类.md §2 红线 3 |
| E7 | **只拍老板自拍碎碎念**，不拍门店作为空间的秩序——属于 `personal_vlog` 或 `founder_ip` 的范畴 | 通用类.md §2 红线 4 |
| E8 | **过度依赖转场和模板**，门店个性被平台模板吃掉 | 通用类.md §2 红线 5 |
| E9 | **把门店日常包装成"今日走单 / 新品播报"纯售卖内容** —— CBNData 指出这是最饱和、最被算法打压的格式 | compass §2 第一段（CBNData 100+ 服装抖音号分析）|

## F. AI 辅助边界

| 环节 | AI 能做 | AI 不能做 |
|---|---|---|
| 选题 | 今日可用的运营动作清单（补货 / 熨烫 / 打包 / 修复）/ 四段式情绪结构建议 / ASMR 声音清单 | 替代"今天门店真实在发生什么"|
| 脚本 | 镜头顺序 / 字幕极简化 / 声音分层设计 / 节奏切点建议 | 替代真实运营流程的偶然性 |
| 拍摄 | 镜头清单 / 构图建议 / 机位清单 / 绑带固定方案（衣架摄像机 / 人台视角）| 替代真实动作和真实声音 |
| 剪辑 | 节奏点 / 蒙太奇建议 / 字幕减法 / BGM 候选（通常建议无 BGM）| 决定"删掉哪些流程、保留哪些痕迹"|
| 发布 | 场景标签 / 时段标签 / 热门 hashtag | 替代"这家店是什么气质"的辨识度 |

> **F 核心判断：** **可深度辅助**——能做选题、声音设计、构图清单、四段式节奏大纲 ; 但**"真实运营痕迹必须来自实拍，AI 不能代演"**。**强烈不建议生成 BGM 密集型或口播型 store_daily**——两者都是 C1 陷阱的特征。
