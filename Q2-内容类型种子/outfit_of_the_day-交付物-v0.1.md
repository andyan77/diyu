## 本文件是什么（核心定义，防止脱离索引被误读）

- **交付物 ≠ 字段 schema**。交付物是一份"**玩法库 + 元素库 + 红线**"，是 AI 生产这种内容时可调用的**表达资源包**。
- **模板 ≠ 文字填空模板**。每张玩法卡里的「30 秒分镜参考」就是模板——**拍摄者照着能拍出来的执行卡片**。
- **AI 生产时不是"填字段"**，而是"**从玩法库里选一张卡片并展开**"。

> 定义来自用户 2026-04-13 裁决。详见 [_index.md](_index.md) §0。

## A. 北极星

**观众看完不只是"这套衣服好看"，而是"我明白这套穿搭在表达什么，我能想象自己穿上以后的状态"——对审美和品牌建立长期记忆。**

> **三份原话备查：**
> - GPT5.4：不只是"这套衣服好看"，而是"我明白这套穿搭在表达什么，我能想象自己穿上以后的状态"
> - 深度研究：产生"我也想这样穿 / 我懂怎么穿了"的冲动，同时建立审美与品牌的长期记忆
> - compass：从"展示搭配"升级为"视觉叙事短片"或"概念叙事"

## B. 元素库（原子层）

### B1 视觉元素
- 全身镜 / 半身特写 / 细节特写（线头 / 纽扣 / 走光点）
- 镜面反射 / 擦拭动作
- 行走 / 坐下 / 拿包 / 卷袖口
- 前后对照造型（通勤/约会/店主/旅行者）
- 分屏（两种身份 / 两种版本）
- 定格动画（衣物"自己穿上身"）
- 俯拍衣物平铺
- 多场景路线（家 → 街 → 店 → 晚餐）

### B2 听觉元素
- 城市环境音（雨声 / 车流 / 吆喝）
- 衣物对比声（防水外套拉链 / 利落鞋跟 / 针织摩擦）
- 通知声 / 脚步声
- 极弱 BGM 或无 BGM

### B3 人物元素
- 独人完成
- 两版本自我（一人双角）
- 关系中的穿着（母女 / 同事 / 朋友）
- 不露脸（只露手 + 衣物）

### B4 场景元素
- 门口 / 玄关 / 试衣镜前（默认）
- 真实路线（通勤 / 店内 / 路上 / 家）
- 反预期场景（地铁站 / 雨后街边 / 凌晨便利店）

### B5 情绪元素
- 决策感（"为什么穿这套"）
- 身份协商（"得体 vs 轻松"）
- 时间痕迹（"早上笔挺，晚上松掉"）
- 情绪翻译（焦虑 / 释然 / 暗涌）

### B6 文字元素
- 字幕只写问题不讲道理
- 关键词字幕（"适合怕热的人"）
- 完全无字
- 默片字幕卡（反差美学）

### B7 时间元素
- 跟随一条真实路线
- 前后对比（穿前 / 穿后被一天使用）
- 多次换装（有因果）
- 30 秒叙事弧光（挑战 → 展开 → 揭晓）

### B8 反差元素
- 得体 vs 轻松（身份谈判）
- 早晨笔挺 vs 夜晚松掉
- 自我视角 vs 他人视角
- 摆拍照片 vs 使用证据

### B9 隐喻元素
- 服装作为"路线适配器"
- 服装作为"身份协商"
- 服装作为"时间痕迹载体"
- 服装作为"情绪翻译器"
- 服装作为"决策结果"

## D. 服装行业垂直机会

OOTD 在服装行业最大的未开发空间，**不是继续卷"搭配公式"**，而是把穿搭拍成：**路线、功能、身份、关系、时间痕迹**。

### 四个未被开发的角度（本 ContentType 因为和服装最相关，角度最多）

1. **"生活适配"（C3-1 今天的路线）**——把 OOTD 从"展示答案"变成"展示生活适配"，这是服装行业最该狠狠干的方向
2. **"时间痕迹"（C3-3 衣服记住了今天）**——天然引出"耐看、耐穿、会呼吸"这类隐性价值
3. **"情绪翻译"（C3-6 情绪衣橱）**——把穿搭升级成情绪叙事，是 AI 最不擅长而人类最有优势的领域
4. **"身份协商"（C3-2 两种身份争同一身衣服）**——把穿搭拍成协商而非答案

### 行业趋势支撑（v0.2 M4 无痕处理：删去具体创作者姓名 / 仅保留方向描述）
- TikTok 2026 说消费者会去 trusted creators 那里验证 real-life product usage
- TikTok 2025 强调 fashion SMB 要靠 authenticity
- **OOTD 叙事化方向** —— 把 OOTD 从"展示穿搭"升级为"视觉叙事短片"（多镜头剪辑 / 故事化结构 / 主题化系列）
- **#stylenotsize 同搭不同体型方向** —— 同一套搭配在不同体型上呈现 / 反对单一身材模板
- **方法论可传播方向** —— 把搭配系统化为简洁可记忆的方法论（如三关键词法 / 三色法 / 单品轮换法）

## F. AI 辅助边界

| 环节 | AI 能做 | AI 不能做 |
|---|---|---|
| 选题 | 场景-穿搭因果链 / 情绪对应建议 / 生活路线清单 | 替代真实生活判断 |
| 脚本 | 问题开头 / 决策过程结构 / 身份协商对话 | 替代"今天为什么这样穿"的真实理由 |
| 拍摄 | 路线分镜 / 细节特写清单 / 对比维度 | 替代真实使用痕迹 |
| 剪辑 | 节奏对齐 / 字幕极简化 / 身份对照剪辑 | 替代"审美一致性"的判断 |
| 发布 | 标题 / 标签 / 封面 | 替代人格 |

> **F 核心判断：** OOTD 最核心的"今天为什么这样穿"，**只能从真实生活里长出来**（GPT5.4）。AI 很适合辅助脚本、出镜顺序、变体方案、字幕精炼，但这个核心不可替代。

## 与试点版（personal_vlog）的差异点

1. **服装行业垂直度最高：** 7 个个人创作类里，OOTD 是和服装行业最直接相关的——几乎每张卡的服装钩子都是 ⭐⭐ 或 ⭐⭐⭐

3. **商业化风险最高：** OOTD 是最容易滑向"硬广 / 种草"的 ContentType，需要最强的红线约束
4. **AI 辅助边界特殊：** "今天为什么这样穿"是最难 AI 化的决策，但一旦穿搭决策做完，后续的分镜 / 字幕 / 剪辑 AI 可以全程辅助
5. **与其他 ContentType 的边界最模糊：** OOTD 和 personal_vlog / daily_fragment / product_review / lifestyle_expression 都有重叠区——这个问题要在 Q2 批量审核时统一处理
