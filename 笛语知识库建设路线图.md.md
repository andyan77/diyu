## 一、月度路线（M0.5 → M6）

### M0.5 · 四个地基（4-6 周 · 不能跳）

> **逻辑**：管道没立起来就采素材，等于把脏水灌进新水库。这一阶段不追求 9 表行数增长，只追求**机制就位**。

| 地基 | 内容 | 验收 |
|---|---|---|
| **地基 1：brand_overlay 扩容立法 v2** | 把 W13 现行 4 类扩到 5 类：`brand_voice` / `brand_persona`（合并原 founder + team_persona）/ `brand_content_type_overlay` / **`brand_event_material`（新增 · 真实事件素材）** / 保留 needs_review 兜底 | `templates/brand_overlay_legislation_v2.md` 落档；G19 硬门更新 |
| **地基 2：raw_material_register（原始素材前置层）** | 真实访谈 / 录音转写 / 朋友圈 / 客户反馈 → 先入 raw register，不直接抽 pack；通过清洗 + 授权 + 评级后才进 source_unit → candidate_pack → 4 闸 → 9 表 | `raw_material_register/register.csv` + `raw_material_taxonomy.md` |
| **地基 3：隐私 / 授权治理 7 字段** | 每条 raw 必填：`consent_status` / `publicity_level` / `anonymization_required` / `commercial_sensitive` / `usable_for_generation` / `usable_for_training` / `source_owner` | 7 字段进 raw register schema；新增 G22 硬门（无授权不得升 pack） |
| **地基 4：evidence_maturity 三级分级** | 取代"单一跨源硬门"：`single_source_candidate`（单源可入但不能 stable）→ `multi_scene_validated`（同品牌多场景）→ `cross_brand_validated`（跨品牌可升稳定通用）| 新增字段 `evidence_maturity`；新增 G23 硬门（domain_general stable 行需 cross_brand_validated）|

**M0.5 不做**：不抽新内容、不补 Q 裁决、不做血肉采集。**只立管道**。

## 四、第三层 · 笛语血肉层（单品牌真实素材）

### 4.1 raw_material_register 流水线（reviewer 意见 3）

```
真实访谈 / 录音 / 朋友圈 / 反馈
      ↓ 入 raw_material_register（必填 7 字段授权）
      ↓ 清洗（去隐私 / 去重 / 切片）
   source_unit
      ↓ 4 闸抽取
   candidate_pack（brand_faye）
      ↓ 投影
   9 tables（brand_layer=brand_faye）
```

**禁止跳级**：raw 未授权不得入 source_unit；source_unit 未签 4 闸不得入 9 表。

### 4.5 隐私授权 7 字段定义（reviewer 意见 4）

| 字段 | 取值 | 说明 |
|---|---|---|
| consent_status | granted / pending / refused / unknown | 当事人是否授权 |
| publicity_level | public / brand_internal / restricted | 可公开度 |
| anonymization_required | yes / no | 是否需脱敏后才能用 |
| commercial_sensitive | yes / no | 是否涉商业机密 |
| usable_for_generation | yes / conditional / no | 能否用于内容生成 |
| usable_for_training | yes / conditional / no | 能否用于模型训练 |
| source_owner | string（如"创始人 Faye"）| 素材所有者 |

**G22 硬门**：consent_status ≠ granted 的 raw 不得升 source_unit。
