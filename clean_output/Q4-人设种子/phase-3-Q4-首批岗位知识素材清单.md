## 二、冷启动阶段的统一采集原则

为避免素材清单失控、空心或采偏，本稿统一采用以下 6 条原则。

### 2.1 先收"岗位真的会发生什么"，不先收"岗位应该说什么"

岗位知识素材优先级必须是：

1. **真实动作**
2. **真实判断**
3. **真实困境**
4. **真实话术**
5. **真实情绪**
6. **再往后才是总结性表达**

否则最容易采到一堆漂亮口号，最后填不出 `authenticity_anchors`（真实性锚点）与 `typical_scenarios`（典型场景）。

### 2.2 一条岗位知识至少要能落到 3 类字段

每条素材至少应能支撑以下 3 类字段中的 2 类以上（字段结构见 [phase-3-内容生产核心思想落盘方案.md:215-263](../phase-3-内容生产核心思想落盘方案.md#L215-L263)）：

- `authenticity_anchors`（真实性锚点）
- `topic_scope`（话题边界）
- `typical_scenarios`（典型场景）
- `emotional_spectrum`（情绪光谱）
- `role_type`（岗位类型）/ 岗位动作层描述

如果一条素材只能形成"好听的一句话"，不能落到这些字段，就不算优质岗位素材。

### 2.3 先收"高频、可复用、可对照"的素材

冷启动阶段优先收：

- 高频发生
- 多门店 / 多品牌能复用
- 能与别的岗位形成对照
- 能稳定支撑首批 `ContentType`（内容类型）

不优先收极端个案、传奇故事、一次性热点。

### 2.4 素材必须带"证据形态"

每条素材至少要标清它属于哪种证据形态：

- **口述类**：访谈、复盘、对话记录
- **动作类**：视频、照片、现场步骤记录
- **文档类**：SOP、工艺单、培训资料、表格、清单
- **物证类**：样衣、吊牌、货架、退货单、纸条、调货记录

没有证据形态，只剩转述总结，可信度会大幅下降。

### 2.5 先收"失败 / 犹豫 / 卡住"，不要只收"做得好"

如果素材全是"标准答案"和"成功案例"，系统会长出很重的 AI 味。岗位知识尤其应该主动收：

- 做砸过什么
- 最怕什么
- 最常卡在哪
- 哪一句最不该说
- 哪一种判断最容易失手

这类素材最容易生成真实的 `authenticity_anchors`（真实性锚点）和 `emotional_spectrum`（情绪光谱）。

### 2.6 一岗至少准备 5 类素材

冷启动阶段，每个 `RoleProfile`（岗位画像）至少要准备以下 5 类素材：

1. **岗位日常动作清单**
2. **岗位判断句 / 经验句**
3. **岗位高频问题与失败案例**
4. **岗位典型场景与时间节点**
5. **岗位特有物件 / 空间 / 证物**

如果还能补到第 6、7 类，会更稳：

6. **岗位对话碎片**
7. **岗位不能说什么 / 不该越界什么**

## 三、首批 7 个岗位的素材清单

### 3.1 `store_manager`（门店店长）

#### 角色定位

挂接 `store_operator`（门店经营者），是门店经营、带队、复盘、培训的主岗位来源。

#### 冷启动必须准备的素材

**A. 岗位日常动作**

- 开店前检查流程
- 闭店后复盘流程
- 顾客高峰期排班 / 顶岗
- 新人带教动作
- 陈列调整 / 补货 / 临时调货
- 处理投诉 / 处理犹豫客

**B. 岗位判断句**

- 什么样的顾客今天更像"来确认"而不是"来逛"
- 什么情况下必须先稳住团队，而不是先冲销量
- 一家店"今天状态不对"最早从哪里能看出来
- 哪些话术 / 动作会让店员前台失手

**C. 高频问题与失败案例**

- 断码、缺货、后场跟不上前场
- 新人接客失手
- 高峰期服务混乱
- 陈列看起来好看但卖不动
- 顾客反复试穿不成交后的复盘

**D. 典型场景与时间节点**

- 开店前 30 分钟
- 中午客流波峰
- 晚高峰前后
- 节假日 / 活动日
- 闭店后复盘

**E. 特有物件 / 空间 / 证物**

- 收银台
- 试衣间
- 调货单 / 缺码记录
- 培训笔记
- 排班表
- 店内镜面与货架状态

#### 推荐证据来源

- 店长访谈
- 闭店后复盘录音
- 门店监控片段（仅作内部知识抽取，不对外）
- 培训记录
- 排班表 / 销售复盘表

#### 首批最少要拿到的 3 个硬证据

- 一次真实闭店复盘记录
- 一次高峰期救火经过
- 一份店长自己的"新人最容易犯错点"清单

### 3.2 `warehouse_supervisor`（仓库主管）

#### 角色定位

挂接 `store_operator`（门店经营者），是后场协同、库存、调货、执行纪律的主岗位来源。

#### 冷启动必须准备的素材

**A. 岗位日常动作**

- 入库 / 出库流程
- 调货流程
- 找货 / 补货 / 返仓
- 断码处理
- 爆款补货优先级判断

**B. 岗位判断句**

- 什么货一看就知道容易压
- 哪种断码最伤门店节奏
- 什么时候要优先保证前台速度，什么时候要先保证准确
- 什么情况下"少一件货"背后不是粗心，而是流程没立住

**C. 高频问题与失败案例**

- 找不到货
- 调货晚了
- 补货节奏跟不上销售节奏
- 仓库账实不一致
- 爆款尺码失衡

**D. 典型场景与时间节点**

- 早上开店前补货
- 中午调货高峰
- 活动日 / 上新日
- 闭店后盘点
- 节假日前备货

**E. 特有物件 / 空间 / 证物**

- 后仓货架
- 箱单 / 调货单
- 尺码标记
- 库存表
- 返仓包裹
- 衣架 / 纸箱 / 封箱台

#### 推荐证据来源

- 仓库主管口述访谈
- 调货记录
- 盘点表
- 后仓现场拍摄
- 爆款补货复盘

#### 首批最少要拿到的 3 个硬证据

- 一次真实调货失误的复盘
- 一份"仓库最怕出现的 5 种情况"清单
- 一段后仓找货 / 补货真实动作视频

### 3.3 `sales_associate`（门店店员 / 导购）

#### 角色定位

挂接 `styling_advisor`（穿搭顾问），是试穿建议、搭配判断、顾客体感翻译、私域转化的核心岗位来源。**对服装零售尤其关键**：这个岗位是 [Q3 新增的 `PrivateOutletChannel`（私域出口位）](../phase-3-Q3-首批平台支持与适配边界.md#L94-L143) 最主要的人设输出源——朋友圈、微信群、私信都靠导购承载。

#### 冷启动必须准备的素材

**A. 岗位日常动作**

- 迎客第一句
- 试衣间建议
- 搭配推荐
- 从"试试看"到"成交"的跟进动作
- 朋友圈内容发法 / 老客触达动作
- 售后跟进 / 回访

**B. 岗位判断句**

- 什么时候顾客是在试探你，不是在拒绝你
- 一套搭配为什么成立
- 什么时候不要再推
- 什么顾客适合先给场景，什么顾客适合先给身体感受
- 私域里哪种内容会让老客反感，哪种内容会让她回你

**C. 高频问题与失败案例**

- 推荐过度导致顾客退缩
- 朋友圈发得太像广告
- 顾客说"再看看"以后断掉
- 试穿体验好但没成交
- 线上问款和线下接待脱节

**D. 典型场景与时间节点**

- 顾客进店第一分钟
- 试衣间前后
- 顾客犹豫不决时
- 朋友圈晚间发圈时段
- 老客回访时

**E. 特有物件 / 空间 / 证物**

- 试衣镜
- 试衣间
- 私域聊天截图（脱敏）
- 朋友圈样例（脱敏）
- 顾客搭配前后照片（脱敏）
- 导购手写推荐单 / 笔记

#### 推荐证据来源

- 导购访谈
- 试衣现场观察记录
- 私域朋友圈样例（脱敏）
- 一对一私信样例（脱敏）
- 成交与未成交案例复盘

#### 首批最少要拿到的 3 个硬证据

- 一次真实试穿建议全过程
- 一份导购自己的"不能这样发朋友圈"反面清单
- 一次老客被重新唤醒的私域触达案例

### 3.4 `designer`（设计师）

#### 角色定位

挂接 `product_professional`（商品专业者），是设计判断、灵感转化、改版逻辑的核心岗位来源。

#### 冷启动必须准备的素材

**A. 岗位日常动作**

- 灵感记录
- 面料挑选
- 版型初判
- 样衣评估
- 修改建议输出

**B. 岗位判断句**

- 什么样的设计看起来好看但上身不成立
- 一件衣服"问题到底出在版型、面料还是细节"
- 为什么要砍掉一个自己很喜欢的方案
- 什么灵感适合品牌，什么不适合

**C. 高频问题与失败案例**

- 想法很好但落不了地
- 样衣上身不成立
- 面料选错导致整体感觉变掉
- 改版太多次
- 设计逻辑和商品逻辑冲突

**D. 典型场景与时间节点**

- 打版前
- 样衣初评
- 改版讨论
- 上新前最后确认
- 面料评审

**E. 特有物件 / 空间 / 证物**

- 灵感板
- 草图 / 纸样
- 面料样卡
- 修改纸条
- 样衣版本对比
- 设计会议记录

#### 推荐证据来源

- 设计师口述访谈
- 草图与样衣版本记录
- 面料选择记录
- 改版意见单
- 设计评审会记录

#### 首批最少要拿到的 3 个硬证据

- 一次样衣不成立的完整复盘
- 一组 1.0 / 2.0 / 3.0 版本对照
- 一段设计师解释"为什么改掉这一版"的原话

### 3.5 `sample_worker`（样衣工）

#### 角色定位

挂接 `product_professional`（商品专业者），是打样、工艺落地、真实劳动感与工艺失败留痕的核心岗位来源。

#### 冷启动必须准备的素材

**A. 岗位日常动作**

- 打样流程
- 裁剪
- 缝制
- 临时返工
- 版师 / 设计要求转化为手上动作

**B. 岗位判断句**

- 哪种设计一看就知道难做
- 哪一步最容易翻车
- 哪种缝法会让成衣质感直接掉下去
- 一件衣服"做出来"和"做得好"差别在哪里

**C. 高频问题与失败案例**

- 车线不顺
- 工艺要求和实际落地冲突
- 样衣比例不对
- 返工太多
- 设计图和成衣效果差距大

**D. 典型场景与时间节点**

- 打样中
- 改版前后
- 样衣返工时
- 赶工时段
- 做到最后发现不对时

**E. 特有物件 / 空间 / 证物**

- 缝纫机
- 线迹特写
- 裁片
- 工艺单
- 返工样衣
- 针线 / 辅料 / 量尺

#### 推荐证据来源

- 样衣工口述访谈
- 打样现场短视频
- 工艺单与样衣对照
- 失败样衣留档
- 返工说明

#### 首批最少要拿到的 3 个硬证据

- 一次返工全过程
- 一组"正确线迹 / 失败线迹"对照
- 一段样衣工讲"这一步最烦、最容易出错"的原话

- 品牌审美边界说明

- 一次"为什么砍掉一个大家都喜欢的方案"的复盘
- 一份设计总监自己的"这个品牌不能长成什么样"清单
- 一段对"品牌边界"最具代表性的原话

---

### 3.7 `merchandising_director`（商品总监）

#### 角色定位

挂接 `product_professional`（商品专业者），是货品结构、上新逻辑、经营视角商品判断的核心岗位来源。

#### 冷启动必须准备的素材

**A. 岗位日常动作**

- 上新节奏安排
- 货品结构判断
- 爆款 / 平销 / 压货分析
- 搭配组合建议
- 商品和门店反馈联动

**B. 岗位判断句**

- 什么款该做深，什么款只能试水
- 为什么一件设计上不错的货在经营上不成立
- 什么样的货适合门店，什么样的货适合线上
- 什么时候应该为门店让步，什么时候应该坚持商品逻辑

**C. 高频问题与失败案例**

- 上新节奏失衡
- 款式结构失衡
- 爆款压不住
- 好看的货卖不动
- 货品判断与一线反馈脱节

**D. 典型场景与时间节点**

- 上新前决策
- 系列搭配判断
- 爆款复盘
- 滞销复盘
- 门店反馈回流时

**E. 特有物件 / 空间 / 证物**

- 商品结构表
- 上新节奏表
- SKU 分析表
- 动销报表
- 门店反馈汇总
- 货品复盘 PPT / 备忘录

#### 推荐证据来源

- 商品总监访谈
- 上新复盘资料
- SKU 表 / 动销表
- 门店反馈汇总
- 爆款 / 滞销案例分析

#### 首批最少要拿到的 3 个硬证据

- 一次"设计很好但商品上不成立"的完整案例
- 一份商品总监自己的"什么货不能做深"的判断清单
- 一段"门店反馈如何改变商品决策"的真实复盘

## 四、首批采集优先级建议

### 4.1 P0（第一轮必须先拿到）

这 4 个岗位优先级最高：

- `sales_associate`（门店店员 / 导购）
- `store_manager`（门店店长）
- `designer`（设计师）
- `sample_worker`（样衣工）

#### 为什么是这 4 个先行

因为它们分别对应：

- 前台接客转化
- 门店经营秩序
- 设计判断
- 工艺落地

这 4 个岗位一旦有了最小素材包，三套通用 `Persona`（人设）就都不会空转：

- `styling_advisor` ← `sales_associate`
- `store_operator` ← `store_manager`
- `product_professional` ← `designer` + `sample_worker`

### 4.2 P1（第二轮补齐）

- `warehouse_supervisor`（仓库主管）
- `design_director`（设计总监）
- `merchandising_director`（商品总监）

#### 为什么放到第二轮

不是因为它们不重要，而是因为冷启动阶段它们更偏：

- 纵深
- 决策
- 协同
- 复盘层价值

对"先把系统跑起来"而言，优先级略低于前台与一线专业岗位。

## 五、统一采集模板（字段占位）

为避免每个岗位素材整理方式不一致，本稿建议每条素材统一记录以下字段：

| 字段 | 含义 |
|---|---|
| `role_profile_id`（岗位画像编号）| 属于哪个岗位 |
| `material_id`（素材编号）| 素材唯一编号 |
| `material_type`（素材类型）| 口述 / 动作 / 文档 / 物证 |
| `scenario_tag`（场景标签）| 发生在哪个典型场景 |
| `evidence_summary`（证据摘要）| 一句话说清素材内容 |
| `can_support_fields`（可支撑字段）| 可用于填哪些字段，如 `authenticity_anchors` / `topic_scope` |
| `risk_note`（风险备注）| 是否有脱敏、权限、品牌边界风险 |
| `source_owner`（素材提供者）| 谁提供 |
| `verification_status`（核验状态）| 待核验 / 已核验 |

**范围说明：** 上述字段是**素材管理内部字段**，不是 `RoleProfile` 实体 schema，不触发 schema 变更。

### 为什么要统一这套采集模板

因为这份文档是"素材清单"，下一份《参数级填充清单：Q4 部分》才是"字段怎么填"。如果这里不先统一最小采集模板，后面会出现：

- 一堆素材进来了，但没法回填字段
- 同一岗位的素材收得很散
- 运营、知识工程、品牌访谈三边口径不一致

## 七、本稿结论

Q4 冷启动阶段，《首批岗位知识素材清单》候选 v0.1 正式收口为：

- **覆盖范围**：首批 7 个 `RoleProfile`（岗位画像）
- **每岗最低要求**：
  - 至少准备 5 类基础素材（日常动作 / 判断句 / 失败案例 / 典型场景 / 物件证物）
  - 至少拿到 3 个硬证据
- **证据形态分类**：统一按"口述 / 动作 / 文档 / 物证"四类采集
- **采集模板**：统一使用 `material_id / material_type / scenario_tag / can_support_fields` 等最小模板记录
- **采集优先级**：
  - **P0（先做）**：`sales_associate` / `store_manager` / `designer` / `sample_worker`
  - **P1（后补）**：`warehouse_supervisor` / `design_director` / `merchandising_director`
