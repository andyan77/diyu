## 本文件是什么（核心定义，防止脱离索引被误读）

- **交付物 ≠ 字段 schema**。交付物是一份"**玩法库 + 元素库 + 红线**"，是 AI 生产这种内容时可调用的**表达资源包**。
- **模板 ≠ 文字填空模板**。每张玩法卡里的「30 秒分镜参考」就是模板——但在本文件里它是"**文案执行骨架**"而不是视频分镜。
- **AI 生产时不是"填字段"**，而是"**从玩法库里选一张卡片并展开**"。

## ⚠️ 本文件的特殊结构：α 改良版字段语义扩展方案（用户 2026-04-14 裁决）

> **跨 CT 规范真源：** 本字段多义标准（「30 秒分镜参考」字段在视频型 CT 与文案型 CT 下两种读法如何选）已于 2026-04-14 升到共享层 [§J.3 字段语义映射标准](_shared-narrative-engines-v0.1.md)，是跨 CT 字段多义的唯一真源。**本段为 `product_copy_general` 的具体落地说明**，正文四段骨架定义保留作为本 CT 的落地细节；跨 CT 规范以共享层 §J.3 为准。

### 1. 为什么 product_copy_general 需要字段语义扩展

前 3 份 batch 3 文件（`store_daily` / `product_review` / `emotion_expression`）和 batch 1/2 的全部 13 份都是**视频型 ContentType** —— 玩法卡的 10 字段中，「30 秒分镜参考」字段是拍摄时的画面 / 声音 / 动作序列。

`product_copy_general` 是**文案型 ContentType** —— 它的输出是详情页 / 海报 / 私聊话术 / 小红书图文笔记，**不是视频**。如果强行填"30 秒分镜"字段会语义错配。

### 2. α 改良版方案（用户 2026-04-14 裁决）

**保留 10 字段结构不拆表，把「30 秒分镜参考」字段的语义在本文件内扩展读作「文案执行骨架」**。具体映射：

| 视频型 ContentType（batch 1/2/3 前 3 份）| 本 CT（product_copy_general） |
|---|---|
| **字段名**：30 秒分镜参考 | **字段名**：30 秒分镜参考（**同槽位不同语义**）|
| **内容**：0-5 秒 / 5-10 秒 / 10-22 秒 / 22-30 秒 四段式分镜 | **内容**：**文案执行骨架四段式** —— ① 钩子标题 / ② 正文节奏 / ③ 视觉分层 / ④ CTA 或留白 |
| **落地**：拍摄者照着分镜拍视频 | **落地**：文案作者照着骨架写详情页 / 海报 / 私聊 |

**核心原则：索引层 10 字段规范不改、不新开模板**。改的是这一份文件内部对"30 秒分镜参考"字段的**解读方式**。batch 1/2 的 13 份和 batch 3 前 3 份全部不动。

### 3. 「文案执行骨架」字段的四个子段

每张 C2 / C3 玩法卡的「30 秒分镜参考」字段**在本文件内必须包含 4 个子段**：

| 子段 | 说明 | 示例（具体化场景替代泛泛夸赞）|
|---|---|---|
| **① 钩子标题** | 首行 / 首句 —— 3-8 秒内决定读者是否读下去 | "早高峰 7:50，手扶地铁杆"|
| **② 正文节奏** | 正文的逻辑序列 —— 像分镜一样有切点 | "→ 袖口不会往上跑 → 会议室冷气刺骨 → 衣领立起来 → 风进不来"|
| **③ 视觉分层** | 配图 / 排版 / 空间层次建议（非视频但仍需视觉承载）| "四张局部图：袖口特写 / 衣领立 / 裤脚特写 / 椅背垂坠"|
| **④ CTA 或留白** | 结尾 —— 行动号召 / 或反向留白 | "不写'立即购买'。写'这件为这些时刻而生'"|

## A. 北极星

**观众看完后，不是"知道这件衣服的参数"，而是"**在脑子里提前穿过一次、犹豫过一次、相信过一次**" —— 文字和视觉一起把 ta 带进了一个**具体的身体感受和使用时刻**，ta 愿意把它加入自己的生活。**

> **三份原话备查：**
> - 通用类.md：用户在没摸到实物前，已经被文字和画面一起带进了一个具体身体感受和使用时刻
> - 通用类compass.md：generate in distinct voice archetypes — 文案不是一种"电商腔"，而是多种不同的主理人声音原型（主理人叙事体 / 诗意留白体 / 工艺叙述体 / 反讽诚实体）
> - 通用类deep-research-report.md：我一下就懂了 / 它解决我哪一个具体场景 / 身体感受 / 身份表达 / 而且我愿意把它加入我的生活

## B. 元素库（原子层）

> **元素库的特殊性**：本 CT 的元素是"**可写进文案的词 / 句式 / 比喻 / 排版**" + "**可作为局部配图的画面**"，而不是"可拍成视频的镜头 / 声音"。B1-B9 的语义在此针对文字型内容做了映射。

### B1 视觉元素（局部配图层）
- 局部动作照（袖口 / 领口 / 口袋 / 裤脚 / 椅背垂坠）
- 面料微距（手指陷进针织 / 衣角落下 / 光下的纹理）
- 物件作为商品替身（吊牌 / 洗标 / 纸袋 / 纽扣 / 价签）
- 白空间海报（高留白 + 局部隐喻 + 一句极短字）
- 生活场景切片（门边穿鞋 / 地铁手扶杆 / 办公室冷气 / 回家椅背）
- 排版分层（标题层 / 正文层 / 证据图层 / 边界声明层）
- **避免**：全身白底大片 / 模特转圈 / 多图拼贴密集卖点 / FABE 模板化排版

### B2 句式 / 节奏元素
- 短句 + 高密度动词
- 反向命令句（"你可能不需要第二件黑外套" / "别买这件除非"）
- 对照句式（"别人看到：XX / 你身体里：YY"）
- 时间戳开场（"下午 5:40 / 早高峰 7:50 / 凌晨下班后"）
- 反问收口但不加感叹号（克制反问）
- 列表式反对意见清单（顾虑 1 / 顾虑 2 / 顾虑 3）
- **避免**："绝绝子 + 冲!!! / 高级 + 百搭 + 显瘦 / 你还在为...烦恼吗 / 今天立减 XX"

### B3 比喻 / 隐喻元素
- **关系型比喻**（"像被穿松了三年的旧 T 恤那种软" / "像第二层肌肤"）
- **拟人化对立**（"让暖和美这对敌人 / 在一条毛呢西裤上和解"——NEIWAI 风）
- **联觉比喻**（"清晨的鸟鸣像一针一线 / 在黎明中缝制新衣"——synesthetic metaphor）
- **关系色彩**（不命名颜色而用关系描述 —— "比黑色更淡一点 / 比纯白更有存在感"）
- **人格化版型**（"廓形就像一位有好奇心的内向者"—— NEIWAI 春季风衣经典文案）
- 服装 = 身体的第二皮肤
- 服装 = 进入社会场景的瞬间接口
- 服装 = 可以"体面地替你活着"的那一层

### B4 感官动词库（本 CT 最重要的词表）
- **触觉类**：塌下来 / 贴住 / 滑过去 / 封住风 / 托住肩 / 陷进去 / 擦过 / 叠起来
- **温度类**：捂住 / 透气 / 凉下来 / 暖起来 / 锁住热 / 漏风
- **重量类**：垂下来 / 轻飘 / 压住 / 落在肩上 / 不坠
- **节奏类**：扫过 / 卡住 / 不顶 / 放下 / 抖开 / 收拢
- **空间类**：够到 / 收紧 / 展开 / 包住 / 挡开

### B5 具体化元素
- 具体时间（"下午 5:40 下楼拿外卖"而非"日常穿搭"）
- 具体动作（"手扶地铁杆"而非"通勤"）
- 具体身体部位（"不扎脖子的高领"而非"舒服"）
- 具体工艺细节（"袖口压线两道 / 口袋口加固一针"而非"品质好"）
- 具体痛点解除（"坐一下午不会一直偷偷扯腰头"而非"久坐舒适"）

### B6 文字载体元素（服装行业独有）
- 洗标文字（"40 度以下温柔对待"）
- 吊牌文字（"适用人群：正在学会拒绝的人"）
- 价签文字（"标价：一次不必解释的舒适"）
- 成分表文字（"成分：70% 忍耐 30% 沉默"）
- 尺码卡文字
- **这些是其他行业完全没有的"文本皮肤"** —— 与 `emotion_expression` C3-4 文本皮肤有交集，但本 CT 的载体服务于"销售"，emotion_expression 服务于"情绪表达"

### B7 排版 / 空间元素
- 高白空间海报（信息密度 ≤ 30% / 留白 ≥ 70%）
- 双栏并置排版（别人看到 / 你身体里）
- 反对意见清单式排版（3 个顾虑 + 3 个证据图）
- 时间戳排版（左侧时间 / 右侧动作）
- 局部 4 格排版（袖口 / 衣领 / 裤脚 / 垂坠）
- 标签文本排版（模拟工业印刷字体）

### B8 声音原型（voice archetypes）—— 来自 compass 的关键概念
- **主理人叙事体**（"我做这件衣服时在想..." 第一人称 / 有创始人性格）
- **诗意留白体**（NEIWAI 风 / 高留白 + 克制比喻）
- **工艺叙述体**（远家 YUANJIA 风 / 技术术语 + 朴素表达）
- **反讽诚实体**（Fashion Brand Company / Reformation 风 / 把缺点先说出口）
- **判断同盟体**（Patagonia "Don't Buy This Jacket" 风 / 反向命令 + 价值观）

### B9 隐喻 / 认知元素
- 文案 = 替代试衣间（在观众脑中完成心理模拟）
- 文案 = 从"评价"到"体验模拟"
- 文案 = 从"对人广播"到"对人说话"
- 文案 = 品牌态度的载体而非卖点的堆叠

| 表达元素清单 | B4 感官动词（不顶 / 扯住 / 贴住）/ B5 具体化（具体时间 / 具体动作 / 具体身体部位）/ B1 局部证据图 / B2 短句节奏 / B8 主理人叙事体 |

| 服装行业钩子 | ⭐⭐⭐⭐⭐ **服装最适合做"具体时刻"销售** —— 因为服装服务的是人进入社会场景的瞬间 —— 这是"事实 + 感受 + 具体时刻"的黄金组合 |
| 实拍可行性 | 🟢 纯文字 + 4 张局部照 / 预算 0 / 4 小时内可批量生成 5-10 条变体 |

| 备注 | 这是本 CT **最可规模化的 AI 辅助玩法** —— 用 B4 感官动词库 + B5 具体化元素，可以为任何一件商品批量生成 10 种具体时刻变体 |

| 字段 | 内容 |

### C3 突破性玩法（笛语平台主推）

> 本档是笛语对 product_copy_general 的核心交付物。**文案突破性不在"更会夸"，而在"更会让观众产生身体想象"**。每张玩法卡的「30 秒分镜参考」字段都是"文案执行骨架"（钩子 / 正文节奏 / 视觉分层 / CTA）。共 5 张。

#### 玩法卡 #C3-1 —— "不是产品文案 / 是一个时刻文案"

| 字段 | 内容 |
|---|---|
| 档位 | 🌟 突破性 |
| **适用场景** | **引流为主**（时刻型文案是第一层引流的强钩子 ; 接客场景时间太短撑不起时刻叙事）|
| 一句话定义 | 整张海报或详情页**首屏**，不写"亚麻衬衫"，而写"**下午 5:40，下楼拿外卖时你懒得换衣服，也不想看起来像刚睡醒**"。图片配的是门边穿鞋的半身 + 袖口和锁骨，**不是标准白底图**。下翻之后才告诉你：这件衬衫为什么适合那个时刻 |
| 北极星映射 | ✅ 不是"这件好看"，而是"**这件懂我会在哪些时刻需要它**" —— 服装按"时刻"卖而不是按"品类"卖 |
| 表达元素清单 | B5 时间戳 / B5 生活切片 / B2 延迟揭示 / B1 局部图 / B2 短句 / B7 留白 |
| 调用共享引擎 | —（本 CT 无共享引擎调用 / 不涉及视频叙事引擎）|
| 30 秒分镜参考（文案执行骨架）| **① 钩子标题**：**不写品类名 / 先写时刻**："**下午 5:40 / 下楼拿外卖**" \| **② 正文节奏**："→ 你懒得换衣服 → 也不想看起来像刚睡醒 → 这件衬衫是你一直穿着的那件 → 下楼时随手拉平就行" \| **③ 视觉分层**：首屏只有一张"门边穿鞋的半身照 + 袖口和锁骨特写" / **下翻之后才出现品类名和面料参数** / 用"延迟揭示"结构 \| **④ CTA 或留白**：不写"立即购买"，写"**这件衬衫为这些时刻而生**" + 时刻清单 |
| 服装行业钩子 | ⭐⭐⭐⭐⭐ **服装是最适合按"时刻"卖的品类** —— 服装服务的是人进入社会场景的瞬间 / 其他品类没有这种"时刻 - 商品"的天然绑定 |
| 实拍可行性 | 🟢 1 人 1 手机拍一张生活图 + 文案 / 预算 0 / 4 小时内完成 |
| AI 辅助度 | ✅ 能做：按服装品类生成"时刻候选清单"（通勤 / 下楼 / 会议 / 约会 / 睡前）、延迟揭示结构模板 ; ❌ 不能做：**验证"这个时刻是不是真实存在的"** |
| 来源 | 通用类.md §3 突破 1 "不是产品文案 是一个时刻文案"（原创设想）|
| 备注 | **这是 α 改良版字段语义扩展的最强验证** —— 玩法的核心就是"**打破先给品类名再给卖点的电商惯性 / 先给生活入口再给产品解释**"。字段语义扩展让"30 秒分镜"天然承载了"时刻文案"的四段式结构 |

#### 玩法卡 #C3-2 —— "双栏文案 / 别人看到的你 / 你身体里发生的事"

| 字段 | 内容 |
|---|---|
| 档位 | 🌟 突破性 |
| **适用场景** | **两者通用**（引流：双栏海报强冲击 ; 接客：店内对犹豫顾客现场展示）|
| 一句话定义 | 海报左边只有一句："**别人看到：干净 / 利落 / 没费力**" ; 右边一句："**你身体里：肩不绷 / 腰不用一直收 / 坐久了也不急着站起来**"。视觉上左边是远景 / 右边是近景触感。**把身体经验正面写进销售语言** |
| 北极星映射 | ✅ 用户第一次会觉得"**这个品牌不是只卖造型 / 它在乎我活在衣服里的状态**"—— 外在形象 + 内在体感的双层价值 |
| 表达元素清单 | B2 双栏排版 / B2 对照句式 / B7 远景 / 近景并置 / B2 低信息量高命中 |
| 30 秒分镜参考（文案执行骨架）| **① 钩子标题**（左右并置）：左"**别人看到**" / 右"**你身体里**" \| **② 正文节奏**：左侧三句"→ 干净 → 利落 → 没费力" / 右侧三句"→ 肩不绷 → 腰不用一直收 → 坐久了也不急着站起来" / **一一对应** \| **③ 视觉分层**：左栏一张远景全身（显外观）+ 右栏一张近景触感（显体感）/ 两张图在同一条水平线 \| **④ CTA 或留白**：不写 CTA / 只写一句"**你现在穿的那件 / 哪一栏占上风?**"|
| 服装行业钩子 | ⭐⭐⭐⭐⭐ **这类写法几乎是服装行业专属** —— 其他品类没有"**外在形象 / 内在体感**"双层价值的这么强 |
| 实拍可行性 | 🟢 **极高**——更像创意排版任务 / 1 人 1 手机 / 预算 0 |
| AI 辅助度 | ✅ **能深度辅助**：按服装品类生成双栏对照句式候选（左"别人看到 X"+ 右"你身体里 Y"）/ 可批量生成 10-20 组变体 ; ❌ 不能做：替代真实的身体体感判断 |
| 来源 | 通用类.md §3 突破 2 "双栏文案"（原创设想）|
| 备注 | **这是 AI 辅助度最高的突破性玩法** —— 因为双栏结构本身就是"**可被模板化但每一栏都必须具体**"的约束 / 正好是 AI 能处理的最甜蜜区 |



#### 玩法卡 #C3-3 —— "反对意见清单体 / 把详情页首屏写成顾虑解除"

| 字段 | 内容 |
|---|---|
| 档位 | 🌟 突破性 |
| **适用场景** | **两者通用**（引流：反向透明 ; 接客：对进店犹豫顾客直接解决顾虑）|
| 一句话定义 | 详情页**首屏**不是品牌自夸 / 而是**三句用户最常见的顾虑**："怕白色太透 / 怕直筒裤坐下卡胯 / 怕针织上身两小时就热"。每句下面**不是大段解释，而是一个很小的实拍证据图或 6 秒短动图**。**不再由品牌定义卖点 / 而是由用户的担忧来组织页面** |
| 北极星映射 | ✅ 观众接住"**你在替我思考 / 而不是替自己说好话**"|
| 表达元素清单 | B2 反对意见 / B1 证据图 / B2 极短文案 / B1 局部动态 / B2 低姿态语气 |
| 30 秒分镜参考（文案执行骨架）| **① 钩子标题**：不是品牌名 / 不是品类名 / 是**三句顾虑**："**你怕白色太透 / 你怕直筒裤坐下卡胯 / 你怕针织上身两小时就热**" \| **② 正文节奏**：每一句顾虑下面**只配一个证据**——"→ 怕透? 这里有逆光测试图" / "→ 怕卡胯? 这里有坐下的实拍" / "→ 怕热? 这里有穿满 2 小时后的汗渍对比" \| **③ 视觉分层**：3 张证据图**绝对不美化** / 就是"逆光测试 + 坐姿 + 汗渍"这种工程师式拍摄 \| **④ CTA 或留白**：不写"立即购买"，写"**如果以上 3 个你都不怕 / 再往下看**"|
| 服装行业钩子 | ⭐⭐⭐⭐⭐ 服装购买的**最大障碍从来不是"用户看不懂审美" / 而是"用户怕踩雷"** —— 这是**服装行业独有的"顾虑图谱"**|
| 实拍可行性 | 🟢 可行 —— 需要的只是把实拍证据组织好 / 预算 0 / 4 小时内完成 |
| AI 辅助度 | ✅ 能做：按服装品类生成顾虑清单（白衬衫 5 顾虑 / 直筒裤 4 顾虑 / 针织 6 顾虑）、证据拍摄清单 ; ❌ 不能做：**决定"这三个顾虑是不是这家品牌真的能回答"** —— 如果回答不了不能用这个玩法 |
| 来源 | 通用类.md §3 突破 3 "反对意见清单体"（原创设想）|

## D. 服装行业垂直机会

服装行业的产品文案**长期有两个极端**：一种是**参数化到像仓库标签**（C1-1 形容词三连 / 尺码参数堆叠），一种是**抒情化到像鸡汤**（C1-2 口号先行 / C1-3 小说体）。**真正的机会在中间** —— **用非常具体的生活时刻 / 身体反馈 / 触感暗示 / 把文字变成替代试穿的界面**。

### 三个未被开发的角度

1. **"具体时刻 + 身体部位 + 触觉动词"的精确化文案**（C2-1 具体场景 / C2-3 触感型 / C2-4 具体化边界）—— 这是**服装行业唯一能做的"替代试衣间"文字空间**，其他品类的文案无法承载如此精确的"摸 / 贴 / 陷 / 塌"触觉映射
2. **"文本皮肤"作为销售载体**（B6 洗标 / 吊牌 / 价签 / 成分表 / 尺码卡）—— 这是**服装行业独占的文字载体**，其他品类只有纸质包装或数字商品描述。与 `emotion_expression` C3-4 文本皮肤的区别是"销售 vs 情绪表达"
3. **"主理人声音原型矩阵"**（B8 主理人叙事体 / 诗意留白体 / 工艺叙述体 / 反讽诚实体 / 判断同盟体 5 种原型）—— 服装行业是**声音原型最丰富的品类** / AI 可以**按品牌性格调用不同声音原型**而不是生成一种"电商腔"

### 服装行业 product_copy_general 标杆案例

| 品牌 / 账号 | 声音原型 | 关键特征 |
|---|---|---|
| **NEIWAI 内外** | 诗意留白体 | "廓形就像一位有好奇心的内向者" / "云朵系列 / 第二层肌肤" / 大量白空间 + 最小文字密度 |
| **远家 YUANJIA**（宁不远 创立）| 工艺叙述体 | "缎纹为底 / 多种彩色丝线为纬 / 八枚经面锻造织造" / 精准裁缝语言 + 口语 "穿上就支棱起来了" |

### 行业趋势支撑

- **Shopify 产品描述规则**：不要停在"excellent quality" / 要写清顾客会在什么场景使用产品 / 产品如何让他们感觉更好
- **Journal of Consumer Research 感官语言研究**：带货语境中使用感官语言会**提升互动与购买意愿** / 机制与"更真实 / 更像真的用过"相关
- **语言具体性研究**：更具体的语言让顾客更满意 / 更愿意购买 / 机制是"顾客推断对方在认真听 / 理解我"
- **视觉隐喻 + 白空间研究**：通过**提升审美愉悦与认知灵活性**影响消费者评价
- **AI 腔文案批判**（compass §3）："**AI 工具正在放大模板问题 —— 以规模生产'高端大气上档次'通用文案** —— 这正是笛语必须用不同方式解决的问题"

## E. 红线（一定不行的玩法）

| # | 红线 | 来源 |
|---|---|---|
| E1 | **抽象形容词堆砌到像机器翻译**："高级 / 显瘦 / 百搭 / 气质 / 绝绝子" —— 观众既不信也不记 | deep-research §product_copy_general 红线 1 |
| E2 | **套促销链条但缺乏证据** —— 会被直接当广告过滤 / 心理防御自动打开 | deep-research §product_copy_general 红线 2 |
| E3 | **把所有信息都塞进一张图** —— 认知负荷高 / 阅读体验差 / 且很像电商模板 | deep-research §product_copy_general 红线 3 |
| E4 | **FABE 模板化详情页** —— Feature-Advantage-Benefit-Evidence 四段式让所有详情页**听起来可互换** | compass §3 "FABE 模板 + 知识诅咒" |
| E5 | **知识诅咒型文案** —— 设计师太了解产品，**无法从消费者视角写** / 写出来的都是内部术语 | compass §3 "Curse of Knowledge" |
| E6 | **反向写法但没有真诚** —— 反销售文案一旦不真诚立刻变成"装高冷的卖货" / 比直接卖货更反感 | 三份原料隐含 + Patagonia 原案例的反面验证 |
| E7 | **把 `emotion_expression` 的 C1 档套进本 CT** —— "允许自己慢一点 + 这件衣服陪你熬过" 是情绪表达的 AI 腔 + 产品文案的 C1 陷阱 / 双重翻车 | 跨 CT 红线 / 本次新增 |
| E8 | **小说体详情页** —— 把一件普通衬衫写成宇宙级人生感悟 / 用户看到第三行就跳出 | 通用类.md §3 主流 3 |

| 环节 | AI 能做 | AI 不能做 |
|---|---|---|
| 选题 | 按服装品类生成"时刻候选清单" / 顾虑图谱清单 / 工艺细节清单 / 触感比喻库 / 声音原型匹配 | 决定"**这个品牌真正的声音是什么**" |
