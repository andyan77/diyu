## 本文件是什么（核心定义，防止脱离索引被误读）

- **交付物 ≠ 字段 schema**。交付物是一份"**玩法库 + 元素库 + 红线**"，是 AI 生产这种内容时可调用的**表达资源包**。
- **模板 ≠ 文字填空模板**。每张玩法卡里的「30 秒分镜参考」就是模板——**拍摄者照着能拍出来的执行卡片**，含画面 + 声音 + 动作 + 节奏，不是"标题 + 正文 + 标签"式的字段拼装。
- **AI 生产时不是"填字段"**，而是"**从玩法库里选一张卡片并展开**"。

> 这个定义来自用户 2026-04-13 裁决，是本目录全部 v0.x 草案的共同前置。详见 [_index.md](_index.md) §0。**2026-04-10 曾因"用 schema 抽象覆盖业务灵魂"翻车，4 份过度工程化文档已归档，本批工作严格使用"玩法卡片化"模型，不退回字段拼装思维。**

## A. 北极星

**观众看完不是"我知道你今天干了什么"，而是"我陪这个人过了一小段真实生活，对他产生真实好感与信任，愿意继续跟着他"。**

> **三份原话备查（合并依据）：**
> - GPT5.4：观众看完以后，不是"我知道你今天干了什么"，而是"我陪这个人过了一小段真实生活，我愿意继续跟着他"
> - 深度研究：观众看完心里像"跟你一起活了十几秒到一分钟"，对你产生真实好感与信任，而不是只记住一段配乐与滤镜
> - compass_artifact：算法在奖励"有灵魂的内容"而非"有套路的内容"；观众接收的应该是"你这个人"，不是"通用漂亮"

## B. 元素库（原子层）

本 ContentType 可调用的所有表达元素，按 9 大类组织。C 段每张玩法卡的"表达元素清单"从这里勾选。

### B1 视觉元素

- 长镜头跟随（第一视角或第三视角）
- 物件特写：口袋 / 袖口 / 褶皱 / 小票 / 钥匙 / 鞋面水渍 / 领口痕迹
- 失焦 / 夜景噪点 / 反光 / 镜头被袖子擦过
- 镜面反射（穿衣镜 / 试衣镜 / 玻璃门 / 水面）
- 第一视角（POV，从门口进入 / 从柜台递物）
- 固定机位 + 走位
- 同构图重复（同一角度不同时刻）
- 分屏（上下两格 / 左右两格）
- 极近景特写（手 / 衣角 / 拉链 / 杯壁水汽）

### B2 听觉元素

- 环境音主导（无 BGM 或 BGM 极弱）
- 衣料摩擦声 / 拉链 / 纽扣 / 鞋跟落地
- 门锁咔哒 / 钥匙串 / 票据撕扯
- 沉默断点（应该有声音的地方突然静默）
- 呼吸声 / 脚步声 / 雨声
- 手机原声收音（不依赖专业麦克风）
- 单句旁白（像备忘录一样的短句，不是完整段落）
- 完全无人声

### B3 人物元素

- 独人（创作者本人）
- 不露脸（只露手、袖口、背影、剪影）
- 第一人称视角 + 微脆弱独白
- 双人同框（与家人 / 同事 / 朋友）
- 第三人称插针（请朋友/家人帮拍 2-5 秒，仍在约束内）
- 镜中自我 vs 他人眼中自我（双视角对照）

### B4 场景元素

- 自家（卧室 / 厨房 / 客厅 / 卫生间 / 阳台）
- 自家店（前台 / 试衣间 / 后仓）
- 真实通勤路线（地铁 / 街边 / 公园 / 便利店）
- 门口 / 门槛（高频出现的"过渡空间"）
- 反预期场景（菜市场 / 老厂房 / 凌晨便利店 / 雨后街边）

### B5 情绪元素

- 真实波动（不解释、不强加）
- 微脆弱表达（"我最近有点怕见人""这周状态回来了 30%"）
- 留白时刻（停顿 / 凝视 / 不说话）
- 情绪曲线（紧→松 / 快→慢 / 焦虑→释然）
- 反高潮（最后什么都没发生）

### B6 文字元素

- 极少字幕（只在关键转折出现）
- 单句旁白（像写给自己的语音备忘录）
- 完全无字幕
- 默片间幕字幕卡（1920 年代风格，仅在转折处）
- 字幕只写感受，不解释事件

### B7 时间元素

- 倒叙（从结果倒推 / 从证物倒推一天）
- 时间跳切（不解释中间发生了什么）
- 同地点 + 不同时刻（早 / 中 / 晚）
- 时间折叠（一周或一天压缩到 60 秒）
- 单一瞬间重复（每天同一个时刻拍 5-7 天）
- 慢镜 / 静帧 / 跳切

### B8 反差元素

- 外部生活 vs 内部状态（行程在动 / 心在停）
- 自我视角 vs 他人视角（同一个人两种凝视）
- 精致镜头 vs 失焦真实
- 主流卡点 vs 突然静音
- 理想中的一天 vs 真实的一天

### B9 隐喻元素

- 衣物作为"时间容器"（被一天使用过的痕迹）
- 衣物作为"身份接口"（同一件衣服不同人的不同读取）
- 衣物作为"情绪皮肤"（紧绷 / 松弛 / 卸载）
- 衣物作为"被生活摩擦过的证物"（口袋掉出小票 / 领口香味 / 鞋面水渍）
- 物件 POV（杯子 / 鞋子 / 椅子作为"见证者"）

### C3 突破性玩法（笛语平台主推）

> 本档玩法是笛语平台的**核心竞争力来源**。AI 生产 personal_vlog 时应该**优先**从 C3 取卡。每张卡片都已通过实拍可行性自检（1 人 + 1 手机 + ≤ 200 元 + 无专业身份依赖）。

#### 玩法卡 #C3-1 —— "一件衣服陪我过完今天"

| 字段 | 内容 |
|---|---|
| 档位 | 🌟 突破性 |
| 一句话定义 | 整条 Vlog 只围绕"一件今天穿在身上的衣服"展开。不解释做了什么，让衣服在不同时段的状态变化自己讲述一天 |
| 北极星映射 | ✅ 观众接住的不是行程，而是陪伴感和具体性："这个人真的活过这一天" |
| 表达元素清单 | B1 衣物特写（口袋/袖口/褶皱）/ B7 时间跳切 / B2 环境音 / B9 衣物作为时间容器 / B6 极少旁白 |
| 30 秒分镜参考 | **0-3 秒：** 早晨门后挂的外套，镜头掠过 \| **3-10 秒：** 出门前你顺手套上 ; 中午挽起袖口，字幕"今天热了" \| **10-20 秒：** 下班地铁里外套搭在腿上 ; 口袋特写掉出一张小票 \| **20-30 秒：** 夜里把衣服挂回门后，那张小票被你顺手抚平，定格 |
| 服装行业钩子 | ⭐⭐⭐ **服装行业最该狠狠干的一条路**——服装在这里不是"展示产品"，而是"做时间容器"，承接情绪、汗意、匆忙、松弛 |
| 实拍可行性 | 🟢 1 人 1 手机可拍 / 预算 0 / 无需专业身份 / 后期仅需按时间顺序拼接 |
| AI 辅助度 | ✅ 能做：分镜清单 / 痕迹清单建议 ; ❌ 不能做：替代真实使用过的痕迹 |
| 来源 | GPT5.4 §1 突破性玩法 1 + 深度研究 §personal_vlog 突破性玩法 1（"衣椅倒叙日记"为变体） |
| 备注 | 与 C3-4 的区别：C3-1 是"正叙陪伴"，C3-4 是"倒叙取证"——可以是同一件衣服的两种叙事手法 |

#### 玩法卡 #C3-2 —— "别人眼里的我，和我自己眼里的我"

| 字段 | 内容 |

## D. 服装行业垂直机会

服装行业做个人 Vlog，**最大的机会不是把 Vlog 当软广插口，而是把"穿着如何参与一天"拍出来**。大多数服装账号只会拍"今天穿什么"，很少拍"这套衣服怎么跟今天的体感、温度、工作、关系发生作用"。

### 三个未被开发的角度

1. **衣服作为"被一天真实使用过的痕迹"** —— 不是新上身那一刻，而是被一天摩擦过之后的样子（褶皱、汗意、口袋掉出的票据、领口的香味）。这是 C3-1 / C3-4 的核心，也是品牌最难伪造的"真实感"
2. **衣服作为"声音媒介"** —— 拉链 / 衣料摩擦 / 鞋跟落地 / 纽扣开合都可以成为听觉品牌资产。这是 C3-7 的核心
3. **衣服作为"关系中的穿着"** —— 在自己眼里是穿搭，在妈妈/朋友/同事眼里是你这个人。服装行业很少有人这样拍。这是 C3-2 的核心

### 行业趋势支撑

- **TikTok 2025 fashion SMB 指南** 强调 authenticity 和 community —— "look behind the curtain" 类内容比 polished perfection 更有说服力（GPT5.4 引 / v0.2 M4 无痕处理：删去原引用品牌名）
- **TikTok 2026 趋势报告** 明确指出 "polished 内容正被 grounding 内容取代"，55% 社交用户更信任人类创作内容（compass 引）
- **2024 小红书趋势关键词** 是"活人感" —— "发疯""抽象"式真实比精致感更受欢迎（compass 引）
- **小红书 2025 电商时尚商家 Playbook** 对"生活分享 / 日常 Vlog / 上身讲解"有明确内容形态线索（深度研究 引）

### 可落地的品牌动作

- 让创作者（品牌主理人 / 店员 / 加盟商）以 C3-1 / C3-4 拍同一件基础款的"陪伴一天" / "倒叙取证"双版本
- 把 C3-7 做成系列："听一件 [品牌] 大衣的一天"
- 把 C3-2 做成"关系中的穿着"主题——母女款 / 情侣款 / 同事款的双视角

## F. AI 辅助边界

| 环节 | AI 能做 | AI 不能做 |
|---|---|---|
| **选题** | 提供话题方向 / 主题清单 / 情绪选项 | 替代真实生活观察 |
| **脚本** | 分镜骨架 / 镜头清单 / 备忘录式短句模板 | 替代创作者的真实情绪与措辞 |
| **拍摄** | 镜头清单 / 节奏点标注 / 服装钩子建议 | 替代实拍现场的偶然性 |
| **剪辑** | 节奏建议 / 字幕精简 / 节拍对齐 | 决定"删掉哪些解释、保留哪些犹豫" |
| **发布** | 标题 / 标签 / 封面建议 | 替代账号人格 |

> **F 核心判断（来自 GPT5.4）：** "YouTube 已经把移动端的粗剪、节拍对齐、模板、AI 初剪这些能力往创作者端推了，但**最后决定这条 Vlog 是否'像人'，仍然是你删掉哪些解释、保留哪些犹豫**。"
>
> 这意味着笛语平台对 personal_vlog 的 AI 生产策略应该是 **"AI 出骨架 + 人出灵魂"**，而不是端到端生成。
