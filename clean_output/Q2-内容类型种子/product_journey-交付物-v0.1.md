## 本文件是什么（核心定义）

- **交付物 ≠ 字段 schema**。交付物是一份"**玩法库 + 元素库 + 红线**"，是 AI 生产这种内容时可调用的**表达资源包**。
- **模板 ≠ 文字填空模板**。每张玩法卡里的「30 秒分镜参考」就是模板——**拍摄者照着能拍出来的执行卡片**。
- **AI 生产时不是"填字段"**，而是"**从玩法库里选一张卡片并展开**"。

> 定义来自用户 2026-04-13 裁决。详见 [_index.md](_index.md) §0。

## 与 `process_trace` 的边界

`product_journey`（本文件）和 `process_trace`（工艺溯源，本批第 5 个）在服装行业容易混淆。边界如下：

| 维度 | product_journey（本文件） | process_trace |
|---|---|---|
| **叙事重心** | **设计决策的历程**（为什么这样设计、为什么改了 3 次、为什么否决那个版本） | **生产工艺的可验证**（为什么这条缝线值得、为什么这块面料经得起检验） |
| **北极星** | "这件衣服不是突然出现在货架上的" | "我愿意相信你没有骗我" |
| **时间** | 从灵感到上市的完整判断链 | 从原料到成衣的物理变化链 |
| **关键证据** | 失败样衣 / 改版纸条 / 样衣对比 | 面料测试 / 缝线特写 / 车间实况 |
| **最核心反套路** | 反"从草图到成品的爽片" | 反"走进工厂的宣传片" |

**生活化例子：** 同样拍一件衬衫——
- 讲"我们为什么把袖口改了 5 次" → `product_journey`
- 讲"这块纯棉面料如何经过 3 次水洗仍不缩水" → `process_trace`

## A. 北极星

**观众看完不是"这是一件突然出现在货架上的衣服"，而是"这是一段被反复推敲、被试错、被等待过的东西——我理解它为什么长这样，并愿意为这个价值付费"。**

> **三份原话备查：**
> - GPT5.4：不是"一件突然出现在货架上的衣服"，而是"一段被反复推敲、被试错、被等待过的东西"
> - 深度研究：我理解这件衣服为什么长这样，把你的产品当作"被认真创造的结果"——叙事心理学的 narrative transportation 机制
> - compass：从"线性流水账"到"一粒纽扣的旅程"——**微物叙事比全景叙事更有冲击力**

## B. 元素库（原子层）

### B1 视觉元素
- 样衣版本树（1.0 / 2.0 / 3.0 并排）
- 失败样衣桌面陈列
- 改版纸条 / 手写修改意见
- 打版纸 / 工艺单 / 质检记录（"纸证据"）
- 手摸面料 / 捏布料 / 拉侧缝 / 试穿
- 桌面固定机位俯拍
- 定格 / 延时（衣服在时间里"慢慢长出来"）
- 倒叙画面（拆线 → 拆版 → 纱线回到纺锤）

### B2 听觉元素
- 剪刀 / 胶带 / 订书机 / 铅笔划过纸张
- 布料摩擦 / 缝纫机 / 面料撕开
- 深夜拉椅子 / 门响 / 叹气
- 语音消息（半句）
- 时间跳切的"不同声音"（每个跳切配不同环境音）
- 无 BGM 或极弱 BGM

### B3 人物元素
- 创作者独人（设计师 / 主理人）
- 双人/多人辩论（老板 + 同事 / 合伙人 / 妈妈 / 店长）
- 只露手
- "隐形的穿着者"（不拍顾客但围绕顾客想象）

### B4 场景元素
- 打版台 / 工作台
- 同一张桌子（贯穿时间）
- 镜子前（上身测试）
- 真实生活场景（坐车 / 上下楼 / 蹲下搬箱子）

### B5 情绪元素
- 犹豫 / 舍不得砍 / 执着
- 焦虑（"做了一个月还没做出来"）
- 争论后的妥协
- 对一个具体顾客的想象（"她早上赶地铁时会不会拉扯肩部"）
- 反"灵感爽片"的代价感

### B6 文字元素
- 版本号（"1.0 / 2.0 / 3.0"）
- 失败原因（"领口高了 1 厘米" / "袖山塌了"）
- 反向缺点开场（"它不够薄 / 它不是最显瘦 / 它不能一衣多穿"）
- 一条真实用户评论作为开场（"通勤裤总是勒肚子"）
- 极少字幕 / 结尾不催单

### B7 时间元素
- 时间跳切（草图 → 纸样 → 边角料 → 未完成样衣 → 被拆版本 → 修改意见 → 最终成品）
- 倒叙（拆线 → 拆版 → 纱线）
- 定格动画（衣服逐步成形）
- 缩时挑战（72 小时做出一件大衣）
- 用户共创连续剧（多集）

### B8 反差元素
- 失败版本 vs 成品
- 保守版 vs 激进版 vs 细节好但成本高版
- "全是好话"的主流 vs "先讲缺点"的反向
- 设计师视角 vs 身体视角 vs 顾客视角

### B9 隐喻元素
- 样衣 = 判断轨迹 / 差一点做成别的样子
- 纸（打版纸 / 工艺单 / 质检记录）= 被系统认真做出来的证据
- 一粒纽扣 = 完整产品历程的微型折射
- 同一张桌子 = 时间养衣服的容器
- 顾客的"想象" = 产品的真正指南针

### C2 进阶玩法（少数高手在用）

#### 玩法卡 #C2-1 —— "样衣版本树 / 原型对比 / 把 3 次失败摆在一面墙上"

| 字段 | 内容 |
|---|---|
| 档位 | ⭐ 进阶 |
| 一句话定义 | 不是只拍最终成品，而是把 1.0、2.0、3.0 的差别摆出来：领口高了 1 厘米、袖山塌了、这个洗后不对、那个上身太像别人家。或把样衣 A/B/C 挂在同一面墙：A 太紧、B 太透、C 肩线不对，镜头逐件近拍问题点 |

### C3 突破性玩法（笛语平台主推）

#### 玩法卡 #C3-1 —— "失败样衣辩论会"

| 字段 | 内容 |
|---|---|

## D. 服装行业垂直机会

`product_journey` 在服装里最强的，不是"可以拍草图"，而是**可以拍犹豫、拍手感、拍身体、拍微差**。服装的每一个决定都能被看见：**缝份、垂感、褶皱、袖口、腰头、里布、洗后状态**。

### 三个未被开发的角度

1. **"让观众看到差一点做成别的样子"（C3-1 失败样衣辩论会）** —— 这是其他行业很少能做的"判断轨迹"展示
2. **"微物叙事"（C3-7 一粒纽扣的旅程）** —— 服装的每个微小元素都能独立成片
3. **"用户共创连续剧"（C3-8）** —— 服装的设计周期长，决策点多，每个决策都可以投票

### 服装行业 product_journey 方向（v0.2 M4 无痕处理：删去全部具体账号 / 品牌 / 数据 / 仅保留方向描述）
- **硬核工艺科普方向** —— 面料 / 版型 / 缝线做工的专业讲解
- **面料对比测评方向** —— 横向对比 + 数据化呈现
- **新中式 / 非遗工艺方向** —— 看似简单的款式背后的工艺难度
- **老牌企业家账号方向** —— 长期稳定的"厂二代 / 传承者"个人 IP 叙事
- **沉浸式大秀 + 用户共创方向** —— 把大秀做成共创连续剧
- **种子用户反馈驱动方向** —— 小批试穿 → 反馈 → 迭代的"产品演化纪实"
